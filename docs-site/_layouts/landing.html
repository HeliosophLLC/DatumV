Why DatumIngest?

SQL Over 7 Formats

Query CSV, JSON, JSONL, ZIP, HDF5, Parquet, and IDX files simultaneously. JOIN across formats in a single query.

ML-Native Types

Vector, Matrix, Tensor, and Image are first-class column types with dedicated functions like softmax, cosine_similarity, and normalize.

Fused Image Pipeline

Chain resize, crop, grayscale, elastic_deform and more. DatumIngest fuses the chain — one decode, one encode.

Source Indexes

Build .datum-index sidecars with chunk statistics, bloom filters, and sorted value indexes for sub-millisecond predicate pruning.

100+ Functions

Math, string, JSON, temporal, ML activations, vector reductions, distance metrics, image transforms — all built in.

Dataset Manifests

One command generates a JSON manifest with per-column statistics, histograms, quantiles, and pairwise column interactions.

Quick Start

Query a CSV file 
datum-ingest explore "SELECT name, score FROM data WHERE score > 90" \
  --source "data=./iris.csv"
Join ZIP + JSON, transform images, output to HDF5 
datum-ingest query "
  SELECT resize_and_crop(load_image(img.file_bytes), 224, 224, 'center') AS image,
         cap.caption
  FROM images AS img
  INNER JOIN captions AS cap ON cap.file_name = GET_FILENAME(img.file_name)
  WHERE len(cap.caption) > 20
  INTO 'dataset.h5'
" --source "images=./train2017.zip" --source "captions=./captions.json"
Sharded ETL with checkpointing 
datum-ingest query "
  SELECT id, normalize(value) AS norm_value, category
  FROM data
  INTO 'output/result.csv' SHARD ON sample_count 10000
" --source "data=./large_dataset.csv" --checkpoint
